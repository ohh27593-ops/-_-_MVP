경북대 밥친구 운영 자동화

경북대학교 학생 대상 밥친구 프로그램을 운영하면서 발생하는 반복 업무를 줄이기 위해 만든 Python 기반 운영 자동화 프로젝트입니다.
이 프로젝트는 구글폼·구글시트 응답을 불러와 가입자 정보를 정리하고, 중복 가입을 방지하며, 탈퇴자 및 비건전 사용자를 반영하고, 신규 사용자 안내와 2차 조사 데이터 정리, 매칭 준비용 파일 생성까지 이어지는 운영 흐름을 코드로 구현한 프로젝트입니다.


1. 프로젝트 개요
학생 대상 밥친구 프로그램을 실제로 운영하다 보면 다음과 같은 반복 업무가 발생합니다.
- 구글폼 응답 확인
- 기존 가입자와 신규 가입자 구분
- 중복 가입 방지
- 탈퇴자 반영
- 비건전 사용자 관리
- 신규 사용자 안내
- 2차 조사 응답 정리
- 질문 및 추가 제안 분리
- 요일별 매칭 준비 파일 생성
이 프로젝트는 위 과정을 수작업이 아니라 Python 코드로 정리하여, 운영자가 더 적은 시간으로 더 안정적으로 프로그램을 관리할 수 있도록 만드는 것을 목표로 했습니다.


2. 프로젝트 목표
이 프로젝트의 목표는 단순히 데이터를 저장하는 것이 아니라, 실제 운영 과정에서 필요한 흐름을 자동화하는 것입니다.
구체적으로는 다음을 목표로 했습니다.
1. 구글시트 설문 응답을 자동으로 불러오기
2. 기존 가입자 정보와 비교하여 신규 가입자만 추출하기
3. 중복 가입을 방지하기
4. 탈퇴자 및 비건전 사용자 정보를 반영하기
5. 신규 사용자에게 안내를 보낼 수 있도록 별도 목록 만들기
6. 2차 조사 데이터를 매칭에 활용할 수 있는 형태로 정리하기
7. 질문, 제안, 요일별 시간대, 선호 활동을 운영에 바로 사용할 수 있는 파일로 저장하기


3. 주요 기능
4. 
3-1. 1차 설문 응답 수집 및 가입자 정보 갱신
- 구글시트에서 1차 설문 응답을 읽어옵니다.
- 응답 시각을 `datetime` 형식으로 변환합니다.
- 이름, 전화번호, 신청 시각 등을 기준으로 가입자 정보를 정리합니다.
- 기존 가입자 파일과 비교하여 신규 가입자만 추출합니다.
- 중복 가입을 방지합니다.
- 탈퇴자 및 비건전 사용자 정보를 반영합니다.
- 
3-2. 신규 가입자 안내 대상 추출
- 새로 가입한 사용자만 따로 저장합니다.
- 운영자가 확인 후 카카오톡 등으로 안내 메시지를 보낼 수 있도록 목록을 만듭니다.

3-3. 카카오톡 안내 자동화
- 신규 사용자 정보를 바탕으로 카카오톡 안내 메시지를 전송하는 기능을 실험했습니다.
- 전화번호 기준으로 신규 대상을 찾고, 전송 여부를 Excel에 기록하는 방식으로 구성했습니다.

3-4. 2차 조사 응답 수집
- 2차 설문 응답 중 실제 참여 의사가 있는 사용자만 추려냅니다.
- 기존 정상 가입자와 대조하여 유효한 응답만 반영합니다.

3-5. 질문 / 제안 분리 저장
- 사용자 질문과 추가 활동 제안을 따로 분리하여 저장합니다.
- 운영자가 사용자 피드백을 따로 확인할 수 있도록 했습니다.

3-6. 매칭 준비용 파일 생성
- 요일별 참여 가능 시간대(오전, 점심, 오후)와 선호 활동(카페, 산책, 맛집, 놀이, 운동)을 기준으로 데이터를 가공합니다.
- 이를 바탕으로 요일별 매칭 준비용 Excel 파일을 생성합니다.


4. 전체 작업 흐름
이 프로젝트의 전체 흐름은 다음과 같습니다.
1. 1차 설문 응답 불러오기
2. 기존 가입자 정보와 비교
3. 신규 가입자 추출
4. 탈퇴자 / 비건전 사용자 반영
5. 신규 사용자 안내 목록 저장
6. 카카오톡 안내 전송 또는 수동 안내
7. 2차 설문 응답 불러오기
8. 참여 의사자만 필터링
9. 질문 / 제안 분리 저장
10. 요일별 매칭 준비 파일 생성

즉, 이 프로젝트는 단순한 설문 정리가 아니라  
가입 → 안내 → 재조사 → 매칭 준비로 이어지는 운영 파이프라인 전체를 코드로 정리한 프로젝트입니다.


5. 사용 기술

- pandas
- numpy
- gspread
- oauth2client
- openpyxl
- pyautogui
